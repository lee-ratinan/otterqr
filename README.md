# OtterQR

## Supported Countries

Currently, the following countries are supported:
- Thailand (PromptPay)

## How to Use

### Thailand PromptPay

#### POST `/generator`

| Key                          | Data Type | Length | Requirement | Description                                                                                                                                                                                                                                                                                                        |
|------------------------------|-----------|--------|-------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `countryCode`                | string    | 2      | M           | For Thailand, the value must always be `TH`                                                                                                                                                                                                                                                                        |
| `pointOfInitiation`          | string    | -      | M           | - For a common QR code without transaction amount, use `STATIC`. <br/> - For dynamically QR code with transaction amount, use `DYNAMIC`. When `merchantAccountInformation` is `BILL_PAYMENT`, this field must be `DYNAMIC`.                                                                                        |
| `merchantAccountInformation` | string    | -      | M           | - For a commercial client using `DYNAMIC` point of initiation with transaction amount and reference number(s), use `BILL_PAYMENT`. <br/> - For common client who uses PromptPay ID (mobile phone number) to receive the payment, use `PROMPTPAY_ID`                                                                |
| `billerId`                   | string    | 15     | C           | **Only valid when `merchantAccountInformation` is `BILL_PAYMENT`.** <br/> The biller ID, 15-digit Tax ID + suffix issued by the financial institution that identifies the beneficiary end of the transaction.                                                                                                      |
| `ref1`                       | string    | max 20 | C           | **Only valid when `merchantAccountInformation` is `BILL_PAYMENT`.** <br/> Reference ID 1. This is the reference ID specific for this transaction. It cannot be reused. Suggestion: This reference ID must be generated by OtterNova system.                                                                        |
| `ref2`                       | string    | max 20 | C/O         | **Only valid when `merchantAccountInformation` is `BILL_PAYMENT`.** <br/> *(Optional)* <br/> Reference ID 2. This is the reference ID specific for this transaction. It cannot be reused. Suggestion: This reference ID can be generated by OtterNova’s client.                                                    |
| `phoneNumber`                | string    | 12     | C           | **Only valid when `merchantAccountInformation` is `PROMPTPAY_ID`.** <br/> The phone number in E.164 format, i.e. `+66123456789` for the number `01 2345 6789`.                                                                                                                                                     |
| `transactionAmount`          | string    | max 13 | C           | **Only valid when `pointOfInitiation` is `DYNAMIC`. <br/> Required when `merchantAccountInformation` is `BILL_PAYMENT`. ** <br/> The transaction amount in THB. Use only numbers and a dot (.) to seperate baht from stang (if any). Only allow 2-decimal points. <br/> E.g. `99` for `฿99`. `89.50` for `฿89.50`. |
| `merchantName`               | string    | max 25 | M           | The merchant’s name.                                                                                                                                                                                                                                                                                               |
| `merchantCity`               | string    | max 15 | M           | The merchant’s city.                                                                                                                                                                                                                                                                                               |
| `postalCode`                 | number    | 5      | M           | The merchant’s postal code                                                                                                                                                                                                                                                                                         |
